# Вклад в проект Flexberry MarkItDown MCP

Спасибо за ваш интерес к участию в проекте Flexberry MarkItDown MCP! Этот документ содержит инструкции по внесению вклада в проект.

## Правила участия

Мы приветствуем вклад от всех! Чтобы сделать процесс внесения изменений максимально простым и эффективным, пожалуйста, следуйте этим рекомендациям.

## Как начать

1. **Форкните репозиторий** - Нажмите кнопку "Fork" на GitHub, чтобы создать копию репозитория в вашем аккаунте.

2. **Клонируйте репозиторий**:
   ```bash
   git clone https://github.com/ВАШ_УЧЕТНЫЙ_ЗАПИСЬ/flexberry-markitdown-mcp.git
   cd flexberry-markitdown-mcp
   ```

3. **Создайте ветку для вашей функции или исправления**:
   ```bash
   git checkout -b feature/название-вашей-функции
   # или
   git checkout -b fix/название-вашей-исправления
   ```

## Руководство по стилю кода

- Следуйте [PEP 8](https://www.python.org/dev/peps/pep-0008/) для Python кода.
- Используйте осмысленные имена переменных и функций.
- Добавляйте комментарии там, где это необходимо, особенно для сложной логики.

## Процесс отправки изменений

1. **Проверьте свой код** - Убедитесь, что ваш код проходит все тесты:
   ```bash
   pytest
   ```

2. **Сделайте коммит** - Коммиты должны быть понятными и содержать описание изменений:
   ```bash
   git add .
   git commit -m "Описание ваших изменений"
   ```

3. **Отправьте изменения**:
   ```bash
   git push origin feature/название-вашей-функции
   ```

4. **Создайте Pull Request** - Перейдите на GitHub и создайте Pull Request из вашей ветки в ветку `main` основного репозитория.

## Шаблоны сообщений коммитов

Мы используем соглашение о сообщениях коммитов:

- `feat:` - Новая функция
- `fix:` - Исправление ошибки
- `docs:` - Изменения в документации
- `style:` - Изменения форматирования (пробелы, отступы и т.д.)
- `refactor:` - Рефакторинг кода
- `test:` - Добавление или изменение тестов
- `chore:` - Обслуживание проекта (обновление зависимостей и т.д.)

Пример:
```
feat: добавлена поддержка конвертации PDF
fix: исправлена ошибка с обработкой кириллицы
docs: обновлен README с новыми инструкциями
```

## Отчеты об ошибках

Если вы нашли ошибку, пожалуйста, создайте issue на GitHub, включив:

1. Краткое описание ошибки
2. Шаги для воспроизведения
3. Ожидаемое поведение
4. Фактическое поведение
5. Окружение (ОС, версия Python и т.д.)
6. Логи или скриншоты, если применимо

## Вопросы и предложения

Если у вас есть вопросы или предложения, пожалуйста, создайте issue на GitHub с меткой `question` или `enhancement`.

## Лицензия

Внося свой вклад, вы соглашаетесь с тем, что ваши изменения будут лицензированы под той же лицензией, что и проект (MIT).

Спасибо за ваш вклад!
